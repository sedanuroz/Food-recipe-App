                                                     FLUTTER PROJECT
Purpose
This application is to provide a comprehensive platform that facilitates the planning and preparation of meals for users. The application aims to make the culinary experience of users more enjoyable by offering various recipes, calorie calculation for healthier choices, personalized note-taking area and the ability to add liked dishes to favorites. Also, with the unique menu creation feature, "What should I cook today?" it makes mealtimes more fun and effortless by putting an end to your troubles. This app is designed to support a balanced and practical lifestyle while encouraging creativity in the kitchen.

Scope
The application consists of 5 basic parts:
-Homepage:
The main page contains basic food categories and allows users to quickly access the content they want.

-Notes Section:
There is a space where you can store the notes you have created yourself. For example, you can prepare your shopping ingredient lists or write your own recipes.

-Calorie Calculation:
There is an area where you can calculate the calories of your recipes for a healthy life. This feature helps you make more informed food choices.

-Favorites Section:
You can easily access the recipes you like by adding them to your favorites later. This feature prevents your favorite recipes from getting lost.

-Menu Builder:
"What should I cook today?" this section, which puts an end to your troubles, recommends to you the menu of the day. Thus, meal planning becomes more enjoyable and effortless.


With this structure, the application aims to make the cooking process of users more practical, organized and fun.

Mandatory Requirements
The application will have a user-friendly interface .It will have five basic parts and will include at least 15 screens. Transitions between screens will be performed Decently using Flutter's MaterialPageRoute feature.
SQLite will be used for data storage, so users will also be able to access their data offline, such as notes, favorite recipes and created menus. This approach will make the user experience faster and more seamless.

Semi-Mandatory Requirements
At a time set by users, a notification will be sent by the app, and users will be encouraged to turn the wheel to create a food menu. This feature aims to provide motivation for meal preparation, while making users' daily meal planning more fun.
This feature both personalizes the user experience and highlights the fun aspect of the application.

User Interface
The application design will be created in accordance with the principles of Material Design to provide a modern and user-friendly experience.
Color Palette: The main colors of the application will consist of pink, green and white tones to create a warm and inviting atmosphere. Pastel shades will be used as auxiliary colors.
Fonts: Plain and modern fonts will be preferred for easy readability, larger and bold fonts will be used for titles, and medium-sized and plain fonts will be used for content.
Icons and Visuals: Minimal and meaningful icons will be selected, appetizing food visuals will be included in the recipes.
Navigation: Clear icons will be used so that users can move comfortably within the application.
Animations: Simple and fluent animations will be added to the application that will add dynamism to the wheel turning feature and screen transitions.









Navigation
The navigation will be configured as follows to allow users to easily switch between screens Decently within the application:

Main Menu Access: Access to the main menu will be provided using a Drawer widget. Users will be able to easily access different categories by clicking on what they want from the icons at the bottom.
Switching Between Pages: Transitions between screens will be made using MaterialPageRoute and visual feedback will be provided to the user during transitions. Decryption will be performed by using the MaterialPageRoute. Decryption.decryption. decryption. decryption. decryption. Thanks to this, users will easily understand which page of the application they are on with each transition.
Animations: Simple animations will be used in screen transitions, especially in features such as turning the wheel and menu suggestions, which will make user interaction more dynamic. These animations will improve the user experience by making transitions more fluid and natural.
Navigation: Navigation between pages will be facilitated by the bottom navigation bar and Decal icons. Thanks to this, users will be able to quickly access all parts of the application.


Technical Infrastructure
Flutter: The application will be developed using Flutter. Flutter's rich widget library will enable the creation of static and dynamic components, creating flexible and customizable designs for the user interface.
SQLite: SQLite database will be used for the storage of user data. Thanks to this, users will be able to securely store their favorite recipes, notes and ingredient lists offline.
Backend Integration (Future Development): Firebase or a similar cloud service can be used in later versions of the application. This infrastructure will be used to synchronize users' data and dynamically update recipes.
Push Notifications: Firebase Cloud Messaging integration will be implemented to send menu creation reminders and other notifications to users at certain times.
Animations and Transitions: With Flutter's animation tools, animations will be created for screen transitions and user interactions, so the application will offer a dynamic and user-friendly experience.







Justifications and Alternative Solutions
Justifications:
Multi-language support will not be included in this project. Because in the first stage of the project, the target audience will be Turkish-speaking users, and language support will add unnecessary complexity. By focusing on the basic functionality of the application, basic features such as recipes, calorie calculation and menu creation, it is more of a priority to simplify the user experience and speed up the project process. This decision will allow the user interface to be simpler and clearer on the project and will speed up the development process. Multi-language support may be added in later versions.

Alternative Solutions:
Instead of multi-language support, users will be able to add their favorite dishes to their favorites. Users can save their favorite dishes with the favorite buttons added under the recipes. This feature allows users to quickly access their favorite recipes, allowing them to create a collection of dishes according to their personal preferences. Thus, not adding multi-language support allows users to use the application more efficiently and in a personalized way, while keeping the project scope simple.
